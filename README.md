## 🎯 О проекте

**Analytics Service** — это микросервисное приложение для сбора и визуализации метрик пользовательского поведения. Проект демонстрирует современный подход к разработке веб-приложений с использованием:

- Асинхронного бэкенда на FastAPI
- Шаблонизации с Jinja2
- Контейнеризации с Docker
- Сбора клиентских метрик через JavaScript

### Основные возможности

- 📈 Сбор просмотров страниц (Page Views)
- 🖱️ Отслеживание пользовательских событий (клики, скроллы)
- 📊 Визуализация метрик на фронтенде
- 🔄 Автоматическая перезагрузка сервисов при сбоях
- 🏥 Встроенные health checks
- 🐳 Полная контейнеризация

## 🚀 Установка и запуск

### Требования

- Docker 24.0.0+
- Docker Compose 2.20.0+
- Git

### Быстрый старт

1. **Клонирование репозитория**
```bash
git clone https://github.com/yourusername/analytics-service.git
cd analytics-service

docker-compose -f devops/docker/compose.yaml up -d --build

# Health check бэкенда
curl http://localhost:8080/health

# Открыть в браузере
open http://localhost:8080

# Остановка всех сервисов
docker-compose -f devops/docker/compose.yaml down

# Просмотр логов
docker-compose -f devops/docker/compose.yaml logs -f

# Перезапуск конкретного сервиса
docker-compose -f devops/docker/compose.yaml restart backend

# Пересборка и перезапуск backend
docker-compose -f devops/docker/compose.yaml up -d --build backend

# Остановка с удалением томов
docker-compose -f devops/docker/compose.yaml down -v
